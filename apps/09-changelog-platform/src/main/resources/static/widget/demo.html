Changelog Widget Demo

Self-contained JavaScript embed widget for changelog platforms

About the Widget

The widget renders a floating bell icon with an unread badge. Clicking it slides in a panel showing the 10 most recent published changelog entries. Entries track read/unread state via localStorage and auto-refresh every 60 seconds.

Quick Start — Local Development

With the changelog platform running on http://localhost:8080, add this script tag to any page:

<script src="http://localhost:8080/widget/widget.js"
        data-slug="my-changelog"
        data-position="bottom-right"></script>

Full Embed Snippet

<script
  src="https://your-cdn.com/widget/widget.js"
  data-slug="your-changelog-slug"
  data-position="bottom-right"
  data-api="https://your-api-host.com">
</script>

Attributes: data-slug (required) — your changelog project slug  |  data-position — bottom-right or bottom-left  |  data-api — override API origin (defaults to widget script origin)

How It Works

  • Bell icon floats at the bottom corner with a red unread badge
  • Clicking opens a 380px slide-over panel from the right (or left)
  • Entries show title, description, type badge, and publish date
  • Unread entries display a blue dot; clicking marks them read (localStorage)
  • Panel polls the API every 60 seconds for new entries
  • All styles are scoped under .changelog-widget-* to avoid conflicts
  • Zero external dependencies — vanilla JS only
Note: The widget below is loaded from the local server. If the server isn't running at http://localhost:8080, the widget will not fetch any entries.
